# 프로젝트: 사주의 길 — 미니사주 풀이 + 사주 매칭 사전등록 웹서비스

## 0. 너의 역할과 작업 방식

너는 이 프로젝트의 풀스택 개발자다. 발주자(나)는 개발자가 아니므로:
- 작업 시작 전 전체 계획을 먼저 보여주고 승인받은 뒤 진행한다.
- 기술적 결정이 필요하면 선택지를 쉬운 말로 설명하고 물어본다.
- 각 단계가 끝나면 내가 직접 눈으로 확인할 수 있는 방법(로컬 실행 명령어 등)을 알려준다.
- 한 번에 다 만들지 말고 아래 단계 순서대로 만든다. 각 단계는 독립적으로 동작해야 한다.

## 1. 서비스 개요

생년월일시를 입력하면 사주 여덟 글자를 뽑아 무료 미니풀이를 보여주고,
두 가지 유료 상품으로 연결하는 깔때기형 웹사이트.

상품 1: 정식 사주풀이 PDF (약 100페이지) — 34,900원 — 결제는 계좌이체 안내 페이지로 연결
상품 2: 사주 궁합 매칭 — 14,900원 — 단, 지금은 결제 없이 "무료 사전등록"만 받는다.
  (등록자 풀에서 궁합 상위 상대가 나타나면 추후 연락하는 구조. 결제는 매칭 성사 시점에 발생.)

브랜드명: 사주의 길
언어: 한국어. 모바일 우선 디자인 (유입의 대부분이 스마트폰).

## 2. 기술 스택 (이대로 고정)

- Next.js (App Router) + Vercel 배포
- 데이터 저장: Supabase (무료 티어). 매칭 사전등록 데이터 저장용.
- 사주 계산: 100% 클라이언트/서버 코드로 직접 구현. 런타임에 AI API 호출 절대 금지.
  모든 해석 텍스트는 빌드 시점에 JSON 파일로 박제되어 있어야 한다. (운영비 0원 원칙)
- 음양력 변환: npm 패키지 korean-lunar-calendar 사용
- 스타일: Tailwind. 외부 유료 서비스 사용 금지.

## 3. 단계별 작업 순서

### 1단계 — 만세력 엔진 (최우선, 가장 중요)

입력: 생년월일(양력 또는 음력 선택), 출생시각(모르면 생략 가능), 성별
출력: 연주/월주/일주/시주의 천간지지 8글자(시주 미상 시 공란), 각 글자의 오행과 음양, 십성 배치, 오행 분포 개수

계산 규칙:
- 연주: 입춘 기준으로 해가 바뀐다 (1월 1일 아님)
- 월주: 절기(절입일) 기준. 절기 날짜는 1940~2050년 데이터 테이블을 코드에 내장할 것
- 일주: 기준일 앵커 방식으로 60갑자 순환 계산
- 시주: 일간에 따른 시두법. 기본은 정자시(자정 기준 날짜 변경, 23시~익일 1시 = 자시).
  야자시/조자시 구분은 설정 옵션으로만 만들어두고 기본 비활성화.
  (확정: 정자시가 서비스 표준. 기준 케이스 1996-06-26 23:38 → 갑오일 갑자시 — 테스트에 포함됨)
- 시각 보정 (확정, PDF 제작 엔진과 동일해야 함): 출생 지역(시/도) 경도 기반 진태양시 보정(균시차 포함)
  적용 후 시진/일주 판정. 서머타임(1987~88) 자동 차감, 1954~1961 UTC+8:30 표준시 처리.
  지역 "모름"이면 경도/균시차 보정 생략. 기본 지역은 서울.
  기준 케이스: 양력 1997-09-29 17:03 부산 여 → 정축년 기유월 갑술일 임신시 (보정 후 16:49 신시) — 테스트에 포함됨.
- 음력 입력은 korean-lunar-calendar로 양력 변환 후 계산. 윤달 입력도 지원.

검증 (반드시 할 것):
- tests 폴더에 테스트 파일을 만들고, 내가 제공할 검증 케이스(실제 만세력 사이트와 교차확인한 생년월일시 → 8글자 정답)를 통과해야 다음 단계로 간다.
- 일단 네가 아는 유명 검증 케이스로 자체 테스트를 먼저 돌려라.

### 2단계 — 해석 데이터베이스 (JSON)

구조만 먼저 설계하고 빈 틀을 만든다. 텍스트 내용은 내가 별도 대화에서 내 톤 가이드 파일과 함께 생성해서 채워넣을 예정이다.

필요한 JSON 구조:
- 60갑자 일주별 풀이 (성격 핵심, 강점, 주의점 — 각 2~3문장)
- 오행별 과다/부족 해석
- 십성 구조별 코멘트 (비겁/식상/재성/관성/인성 중 가장 강한 것 기준)
- 시주 미상일 때 안내 문구

각 항목에 id를 부여하고, 엔진 출력 결과에서 해당 텍스트를 조립하는 함수까지 만든다.

### 3단계 — 사이트 본체

페이지 구성:
1. 메인/입력 페이지: 생년월일시 입력 폼. 시 모름 체크박스. 음력 선택. 성별.
   출생 지역 선택(시/도 단위 드롭다운, 기본값 서울, "모름" 선택 가능) — 진태양시 보정에 사용.
2. 결과 페이지:
   - 8글자를 사주 명식표 형태로 시각화 (모바일에서 보기 좋게)
   - 오행 분포 간단 차트
   - 미니풀이 텍스트 (2단계 JSON 조립)
   - 그 아래 두 개의 CTA 카드:
     a) "나의 정식 사주풀이 받기 — 34,900원" → 결제 페이지(계좌이체 안내)로 이동
     b) "내 사주와 가장 잘 맞는 인연 찾기 — 무료 사전등록" → 매칭 등록 폼으로
2-1. 결제 페이지 (계좌이체 안내) — PDF 구매 및 (추후) 매칭 결제 공용:
   - 상품명과 가격 표시
   - 입금 계좌 표시: 카카오뱅크 3333-32-0545057 문*인구(사주의길)
   - 계좌번호 복사 버튼
   - 안내 문구: "입금 후 카카오톡 채널로 입금자명을 알려주시면 신청서를 보내드립니다" + 카카오톡 채널 링크 버튼
   - 계좌번호와 카카오톡 채널 링크는 env 변수로 관리 (발주자가 직접 변경 가능하게)
   - 현금영수증 관련 문구는 넣지 않는다
3. 매칭 사전등록 폼: 결과 페이지에서 계산된 사주 데이터를 자동으로 가져오고, 추가로 닉네임, 성별, 출생연도, 거주 지역(시/도 단위), 연락처(카카오톡 ID 또는 전화번호), 개인정보 수집 동의 체크. 제출 시 Supabase에 저장.
4. 개인정보처리방침 페이지 (표준 템플릿 기반으로 작성, 수집 항목/목적/보관기간 명시)

디자인 방향: 점집 느낌의 촌스러움 금지. 차분하고 신뢰감 있는 톤(딥네이비/먹색 + 금색 포인트 정도), 전통 요소는 절제해서.

### 4단계 — 궁합 매칭 알고리즘

두 사주를 받아 0~100점 궁합 점수를 내는 함수:
- 일간 천간합 여부 및 상생/상극 관계
- 일지 관계: 육합/삼합 가산, 충/형/해 감산
- 오행 보완성: 한쪽의 부족 오행을 상대가 채워주는 정도
- 가중치는 코드 상수로 분리해서 내가 조정 가능하게

관리 기능은 따로 만들지 말 것. 등록자 조회와 매칭 점수 계산은 Supabase 대시보드 + 간단한 스크립트(전체 등록자 대상 상위 궁합 출력)로 충분하다.

## 4. 절대 규칙

- 런타임 AI API 호출 금지 (운영비 0원)
- 등록자의 생년월일시와 연락처는 민감정보다. 클라이언트에 다른 사람 데이터가 절대 노출되지 않게 할 것 (Supabase RLS 설정 포함)
- 결제 기능(PG/카드결제)을 직접 구현하지 말 것. 계좌이체 안내 페이지만 만든다.
- 모든 화면 문구는 한국어, 존댓말, 과장 없는 톤.

## 5. 시작 지시

지금부터 1단계 만세력 엔진을 시작해라. 먼저 프로젝트 초기 세팅 계획과 1단계 구현 계획을 요약해서 보여주고, 내 승인 후 진행해라.
